# PRD — DataMonitor PoC

Product Requirements Document  
버전: 1.0  
작성일: 2026-05-08

---

## 개요

DataMonitor는 JSON 파일에 저장된 주문 및 시료 데이터를 콘솔 화면에서 조회하고 상태를 모니터링하는 C++ 콘솔 기반 관리자 도구이다.

---

## 기능 요구사항

### FR-01. 전체 Dashboard 표시

- 주문 상태별 집계 결과와 시료 재고 현황을 하나의 화면에 표시한다.
- 화면은 구분선으로 두 영역을 시각적으로 분리한다.
- 데이터가 없을 경우 각 영역에 "데이터 없음" 메시지를 표시한다.

### FR-02. 주문 상태별 집계

- `data/orders.json`을 읽어 전체 주문을 로드한다.
- `status` 필드를 기준으로 4가지 상태로 분류한다.

  | 상태 | 설명 |
  |---|---|
  | 접수 | 신규 접수된 주문 |
  | 처리 중 | 진행 중인 주문 |
  | 완료 | 처리 완료된 주문 |
  | 취소 | 취소된 주문 |

- 각 상태별 건수와 전체 합계를 표시한다.
- `orders.json`이 없을 경우 모든 건수를 0으로 표시한다.

### FR-03. 시료 재고 현황 조회

- `data/samples.json`을 읽어 전체 시료 목록을 로드한다.
- ID, 이름, 설명을 표 형식으로 표시한다.
- ID 오름차순(파일 저장 순서)으로 표시한다.
- 총 건수를 하단에 표시한다.
- `samples.json`이 없거나 데이터가 없을 경우 "등록된 시료가 없습니다." 메시지를 표시한다.

### FR-04. 모니터링 서브메뉴 제공

- 메인 메뉴의 "모니터링" 항목 선택 시 서브메뉴로 진입한다.
- 서브메뉴 항목:
  - 1. 전체 Dashboard
  - 2. 주문 상태별 집계
  - 3. 시료 재고 현황
  - 0. 돌아가기
- 잘못된 입력 시 오류 메시지를 출력하고 메뉴를 재표시한다.

### FR-05. Seed 데이터 지원

- `orders.json`이 없을 경우 빈 초기 파일(`nextId: 1, items: []`)을 자동 생성한다.
- `samples.json`이 없을 경우 동일하게 자동 생성한다.

## 비기능 요구사항

### NFR-01. 플랫폼

- 운영체제: Windows 10/11 (64-bit)
- 컴파일러: MSVC (Visual Studio 2022), C++17 이상

### NFR-02. 데이터 소스

- JSON 파일만 사용한다.
- 외부 네트워크 연결 없이 로컬 파일만 참조한다.

### NFR-03. 읽기 전용

- 모니터링 기능은 데이터를 읽기만 한다.
- 파일 수정, 삭제, 추가는 모니터링 기능 범위에 포함되지 않는다.

### NFR-04. 응답성

- JSON 파일 로드 및 집계는 파일 크기 1MB 미만 기준으로 체감 지연 없이 완료되어야 한다.

### NFR-05. 오류 처리

- JSON 파일이 없을 경우: 빈 데이터로 처리하며 프로그램이 종료되지 않는다.
- JSON 파일이 손상된 경우: 손상 파일을 `.bak`으로 백업하고 빈 데이터로 처리한다.
- 예외 발생 시 사용자에게 오류 메시지를 출력하고 메뉴로 복귀한다.

### NFR-06. 코드 구조

- MVC 패턴을 준수한다.
- 계층 간 의존성은 인터페이스(추상 클래스)를 통해 주입한다.
- 집계 로직은 Controller나 View가 아닌 Service 레이어에 위치한다.

## 모니터링 기준

### 주문 상태 집계 기준

- `Order.status` 필드의 정확한 문자열 값으로 분류한다.
- 허용 상태값: `접수`, `처리 중`, `완료`, `취소`
- 허용 외 상태값이 존재할 경우: 집계에서 제외하고 별도 카운트하지 않는다.
- 상태 비교는 대소문자 구분(exact match)으로 처리한다.

### 시료 재고 현황 기준

- `samples.json`의 `items` 배열 전체를 재고로 간주한다.
- 별도 수량(quantity) 필드 없이 건수 기준으로만 표시한다.
- 시료 삭제/추가는 모니터링 범위가 아니므로, 파일에 저장된 데이터를 그대로 표시한다.

### 데이터 갱신 기준

- 메뉴 항목을 선택할 때마다 JSON 파일을 다시 읽어 최신 상태를 반영한다.
- 자동 갱신(polling) 기능은 이번 PoC 범위에 포함하지 않는다.
